# ArchUnitTS Documentation Assets

This folder contains custom styling and assets for the ArchUnitTS documentation generated by TypeDoc.

## Files

### `custom.css`

The main stylesheet that applies modern website styling inspired by Cursor, Parcel, and contemporary web design to the TypeDoc-generated documentation.

## Design System

### Color Palette

The design uses a modern, accessible color palette inspired by Cursor's interface and modern web standards:

- **Primary Background**: `#fafafa` (Clean neutral base)
- **Secondary Background**: `#ffffff` (Pure white for cards)
- **Tertiary Background**: `#f5f5f5` (Subtle gray for interactive elements)
- **Accent Color**: `#6366f1` (Modern indigo for primary actions)
- **Success Color**: `#10b981` (Green for positive states)
- **Warning Color**: `#f59e0b` (Amber for caution)
- **Error Color**: `#ef4444` (Red for errors)
- **Text Primary**: `#0f172a` (Deep slate for main text)
- **Text Secondary**: `#475569` (Medium slate for secondary text)

### Typography

- **Primary Font**: Inter, SF Pro Display (modern system fonts)
- **Code Font**: JetBrains Mono, Fira Code, SF Mono, Cascadia Code (premium monospace)
- **Font Weights**: 400 (regular), 500 (medium), 600 (semibold), 700 (bold), 800 (extrabold)
- **Line Height**: 1.7 for optimal readability

### Key Features

1. **Modern Card-Based Layout**

   - Larger corner radius (`8px`, `12px`, `20px`) for contemporary look
   - Subtle shadows with proper depth perception
   - Clean spacing following 8px grid system

2. **Gradient Hero Section**

   - Multi-color gradient (indigo to purple to cyan)
   - Grid pattern overlay for texture
   - Large, bold typography with negative letter spacing

3. **Enhanced Interactive Elements**

   - Smooth micro-interactions (0.15s timing)
   - Color-coded element types with rounded icons
   - Hover states with proper feedback

4. **Premium Code Presentation**

   - Dark theme code blocks with better contrast
   - Language badges on code signatures
   - Monospace font stack optimized for code

5. **Responsive Grid System**

   - Breakpoints at 768px and 1024px
   - Flexible sidebar that stacks on mobile
   - Container max-width of 1400px

6. **Architecture-Specific Branding**

   - Color-coded signatures for ArchUnit concepts:
     - Rules: Red (`#ef4444`)
     - Layers: Green (`#10b981`)
     - Slices: Indigo (`#6366f1`)
     - Metrics: Amber (`#f59e0b`)

7. **Modern Table Design**
   - Separated borders with rounded corners
   - Proper spacing and typography hierarchy
   - Hover states for better UX

## Architecture

The CSS follows modern standards and is organized into logical sections:

1. **CSS Custom Properties** - Modern color system and design tokens
2. **Global Styles** - Typography and base layout using modern font stacks
3. **Component Styles** - Card-based components with proper shadows
4. **Interactive States** - Micro-interactions and hover effects
5. **Responsive Design** - Mobile-first breakpoints
6. **Dark Mode Support** - System preference detection
7. **Architecture Branding** - Domain-specific color coding

## Modern Features

- **CSS Grid & Flexbox**: Modern layout techniques
- **Custom Properties**: Dynamic theming system
- **Backdrop Filters**: Modern visual effects (where supported)
- **Container Queries**: Future-ready responsive design
- **Logical Properties**: International layout support

## Usage

The custom CSS is automatically applied to TypeDoc-generated documentation through the `customCss` option in `typedoc.json`:

```json
{
  "customCss": "docs-assets/custom.css"
}
```

## Maintenance

When updating the styling:

1. Follow the modern CSS custom property system
2. Maintain WCAG AA contrast ratios for accessibility
3. Test across light and dark modes
4. Ensure responsive behavior follows mobile-first approach
5. Keep the ArchUnit domain-specific styling consistent with the overall modern design system
6. Use modern CSS features like logical properties and container queries where appropriate
